# Contribution Guide

This document is a concise guide for contributing to the project.

## 1) Environment setup

```bash
npm install
npm dev
```

## 2) Branch strategy

- Create a dedicated branch for each task.
- Suggested naming:
  - `feature/<short-description>`
  - `fix/<short-description>`
  - `refactor/<short-description>`

## 3) Coding principles

- Keep changes small and focused.
- Do not introduce unrelated refactors.
- Preserve the existing file style.
- Prefer updating the proper model over weakening TypeScript types.
